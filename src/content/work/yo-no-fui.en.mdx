title: "Yo No Fui"
year: 2025
category: ["ai", "frontend"]
summary: "A mystery and deduction game: you are the detective and solve cases generated dynamically by AI, with different difficulty levels."
problem: "A detective game with hand-written cases runs out quickly: once you know the solution, the fun is gone. AI promises infinite cases, but it generates text that can be incoherent — clues that lead nowhere, a culprit who contradicts the evidence — and that breaks the game's logic. The challenge was to use AI for variety without the case ever becoming unsolvable."
role: "Solo frontend development: islands architecture, AI case generation, validation of the model output, game state and the whole investigation UI. Deployment included."
decisions:
  - decision: "Validate the model output against a schema before letting it enter the game state."
    rationale: "A model can hallucinate an incoherent case (suspects with no clues, a culprit who isn't listed). Instead of trusting the raw text, the response is validated and normalized against an expected shape; if it doesn't fit, it is discarded and requested again. That way the player never gets an unsolvable case."
  - decision: "Astro 5 with React 19 islands only on the interactive parts (interrogation, board), not the whole page."
    rationale: "Most of the screen is content that needs no JS. Hydrating only the islands that react to interaction keeps the JavaScript shipped small and the load fast, while the game logic lives where it's actually needed."
  - decision: "Centralize game state in Zustand instead of passing it through props between islands."
    rationale: "Case state (revealed clues, interrogated suspects, progress) is shared by several independent islands. A small shared store avoids prop-drilling between islands that aren't in a parent-child relationship and keeps the game's truth in a single place."
challenges:
  - "Making every generated case coherent and solvable: schema validation filters out responses that don't hold together as a case."
  - "Sharing state across independent Astro islands with no common component tree: solved with an external store (Zustand)."
  - "Scaling difficulty without rewriting the logic: levels adjust the case parameters, not the game engine."
result: "Yo No Fui is a deployed game where every session is a new AI-generated case that is always solvable. It demonstrates a responsible use of generative AI — validated, not blindly trusted — over a lightweight islands architecture."

Yo No Fui is a detective game where each case is generated by AI, so it never repeats.
The interesting part wasn't asking the model for cases, but making sure each case *holds
together*: that the clues lead to a culprit and the culprit doesn't contradict the
evidence. So the AI proposes, but a schema decides what enters the game.
